The Architectural Ledger is a modern, comprehensive Human Resources Management Dashboard designed to streamline employee administration, centralize departmental oversight, and manage payroll with precision. Built using a robust, highly-responsive technology stack, it provides an intuitive architecture that scales alongside your organization.

## ✨ Features

- **🛡️ Secure Authentication**: Full-featured login and registration system.
- **📊 Interactive Dashboard**: A unified command center with data visualizations and quick insights.
- **📁 Employee Directory**: Browse, filter, and manage organizational members.
- **👤 Detailed Employee Profiles**: Comprehensive insights into individual employee data.
- **🏢 Department Management**: Organize teams, oversee departmental structures, and assign roles.
- **💰 Payroll System**: Oversee compensation, generate reports, and manage financial ledgers.
- **⚙️ Settings & Configuration**: Customizable preferences for tailored user experiences.

## 🛠️ Technology Stack

- **Frontend Framework**: [React 19](https://react.dev/) with [Vite](https://vitejs.dev/)
- **Routing**: [React Router](https://reactrouter.com/)
- **Styling**: [Tailwind CSS v4](https://tailwindcss.com/)
- **State Management & Context**: React Context API
- **Backend & Database**: [Firebase](https://firebase.google.com/)
- **Charting**: [Recharts](https://recharts.org/)
- **Icons**: [Lucide React](https://lucide.dev/)

## 🚀 Getting Started

Follow these steps to set up the project locally:

### Prerequisites

Ensure you have Node.js (v18+) and npm installed on your machine.

### Installation

1. **Clone the repository:**
   ```bash
   git clone https://github.com/DeekshaG96/the-architectural-ledger.git
   cd the-architectural-ledger
   ```

2. **Install dependencies:**
   ```bash
   npm install
   ```

3. **Environment Setup:**
   Create a `.env` file in the root directory based on `.env.example` and add your Firebase configuration and any necessary API keys.

4. **Start the development server:**
   ```bash
   npm run dev
   ```
   *The application will optionally run on `http://localhost:3000` depending on the configuration.*

## 🏗️ Project Structure

- `src/components/`: Reusable, modular UI components.
- `src/pages/`: Core application views (Dashboard, Payroll, Directory, etc.).
- `src/context/`: Global application state management (e.g., AuthProvider).
- `src/lib/` & `src/constants/`: Utility functions and global constants.
